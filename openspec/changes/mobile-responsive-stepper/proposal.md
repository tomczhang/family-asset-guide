## Why

当前应用仅为桌面端设计（固定右侧 260px 目录栏 + 760px 内容区），在移动端无法正常使用。需要添加移动端响应式支持，将四个编辑区块（资产清单、账户访问与 2FA、五阶段 SOP、自定义区）转化为分步页面（stepper），每步占一整页，用户逐步填写，体验更清晰。目录栏在移动端默认收起，点击后展开作为导航面板，选择步骤后跳转对应页面。

## What Changes

- 新增移动端断点检测（≤768px 触发移动布局）
- 移动端下四个编辑区块变为分步页面，底部有"上一步/下一步"导航按钮
- 移动端目录栏（TableOfContents）默认隐藏，通过浮动按钮打开，展开为覆盖层面板
- 目录面板中点击某步骤直接跳转到该页面，同时面板自动关闭
- 移动端 Toolbar 按钮收缩为菜单或缩小展示
- 桌面端行为完全保持不变

## Capabilities

### New Capabilities
- `mobile-stepper`: 移动端分步浏览体验，包括步骤导航、页面切换、浮动目录按钮和覆盖层导航面板
- `responsive-layout`: 响应式布局断点处理，移动端隐藏固定侧边栏、调整主内容区宽度和间距

### Modified Capabilities

（无现有 spec 需要修改）

## Impact

- **组件改动**: `App.tsx`, `TableOfContents.tsx/.css`, `Toolbar.tsx/.css`
- **样式改动**: `global.css`（布局变量和媒体查询）
- **新增组件**: 可能需要 `MobileStepper.tsx` 管理步骤状态和导航
- **状态管理**: 需要新增当前步骤索引状态（仅移动端使用）
- **无依赖变更**: 不需要新增 npm 包
