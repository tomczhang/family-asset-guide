## Context

当前资产录入流程要求用户手动输入机构名称、登录网址、联系电话。但这些信息对于已知的金融机构是固定的公开信息。项目是纯前端离线应用（Vite + React），无后端 API。

## Goals / Non-Goals

**Goals:**
- 用户选择机构后自动填充官网、电话、APP 下载链接
- 预设数据内嵌在前端代码中，保持离线可用
- 用户仍可选择"其他"并手动输入自定义机构信息
- 支持按资产类型筛选可选机构

**Non-Goals:**
- 不做机构数据的在线更新/同步
- 不做机构 logo 或品牌视觉展示
- 不做机构搜索的模糊匹配（当前规模用简单下拉即可）

## Decisions

### 1. 数据结构：静态 TypeScript 对象

机构预设数据以 TypeScript 常量形式存放在 `src/data/institutions.ts`，按资产类型分类。

理由：项目是纯前端离线应用，数据量可控（预计 30-50 家机构），TS 常量享受类型安全、tree-shaking、零运行时开销。

### 2. 机构选择交互：下拉选择 + "其他"选项

用户先选资产类型（已有），再选该类型下的机构。选择机构后自动填充 loginUrl、contactPhone、appDownload。选择"其他"时显示手动输入字段。

理由：机构数量可控，下拉比搜索框更简单直观；保留"其他"确保灵活性。

### 3. Asset 数据模型调整

- 新增 `institutionId: string`（对应预设机构 ID，选"其他"时为空串）
- 新增 `appDownload: string`（APP 下载方式/链接）
- `institution`、`loginUrl`、`contactPhone` 保留为自定义覆盖字段（选预设机构时自动填充但用户可编辑）

理由：保持向后兼容，旧草稿文件中无 institutionId 的资产仍可正常显示。

### 4. 预设数据覆盖策略

选择预设机构时，将预设值写入 institution/loginUrl/contactPhone/appDownload 字段。用户可以在此基础上修改。切换机构时重新填充（覆盖当前值）。

理由：简单直观，用户看到的就是实际存储的值，无需区分"预设值"和"用户值"。

## Risks / Trade-offs

- [机构信息过时] → 用户可手动修改任何字段；后续版本可通过更新应用来更新预设数据
- [预设机构覆盖面不足] → "其他"选项保底；可逐步扩充
- [切换机构覆盖用户修改] → 可接受，切换机构本身意味着信息需要重填
