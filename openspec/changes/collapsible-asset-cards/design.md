## Context

AssetCard.tsx 当前所有资产卡片始终展开。随着资产增多，页面过长。需要只展开正在编辑的那张卡片。

## Goals / Non-Goals

**Goals:**
- 同一时间只有一张卡片处于展开编辑状态
- 收起的卡片以紧凑摘要行展示：类型 | 机构 | 账号 | 估值
- 添加新资产时自动展开新卡片、收起旧卡片
- 收起卡片右侧有"编辑"+"删除"操作按钮

**Non-Goals:**
- 不做动画过渡（保持简单）
- 不做多卡片同时展开

## Decisions

### 状态管理：组件内 useState

用一个 `expandedId: string | null` state 控制当前展开的资产 ID。添加资产时通过 dispatch 回调后将新资产 ID 设为 expandedId。

理由：这是纯 UI 状态，不需要持久化到 document 中。

### 摘要行内容

收起时显示：`[资产类型标签] [机构名称] [账号(脱敏)] [币种+估值]`，一行紧凑展示。

### 交互规则

- 点击"编辑"→ 展开该卡片，收起其他
- 点击"添加资产"→ 新建卡片自动展开，其余全部收起
- 没有单独的"收起"按钮（切换到其他卡片时自动收起）

## Risks / Trade-offs

- [新建资产 ID 需要在 dispatch 后获取] → 改造 ADD_ASSET action 返回 ID，或在组件中预生成 ID
